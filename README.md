# 내 자산 대시보드

미래에셋(주식) + KB(예적금)를 한 화면에서 보는 대시보드.
GitHub Pages 에 올리고 노션에 `/embed` 로 삽입한다.

## 개인정보는 이 레포에 없다

보유수량·매입단가·적금 잔액은 **노션 embed URL 의 `#c=` 해시에만** 존재한다.
브라우저는 해시(`#` 뒤)를 서버로 전송하지 않으므로 GitHub 은 그 값을 볼 수 없다.
이 레포에 있는 것은 티커 목록과 공개 시장 데이터(시세·환율)뿐이다.

## 왜 이 구조인가

브라우저에서 Yahoo·stooq 를 직접 부르면 **CORS 로 전부 차단된다**(확인함).
그래서 GitHub Actions(서버)가 yfinance 로 시세를 받아 `data/*.json` 으로 커밋하고,
페이지는 같은 출처의 JSON 만 읽는다. → CORS 없음, API 키 없음, 공짜.

```
Actions(cron 30분) ──yfinance──> data/prices.json   (현재가·환율)
                                 data/history.json  (일별 종가 400일)
                                        │
노션 embed URL #c=…(보유·적금) ─────────┴──> index.html ──> 화면
```

## 매달 하는 일

노션 embed URL 의 `#c=` 값만 갈아끼운다. (Claude 에게 새 보유 내역을 주면 URL 을 만들어 준다.)
종목을 새로 사면 `tickers.json` 에 티커도 추가한다.

## 추이 그래프의 한계

현재 **매매 이력이 없어서**, 추이 곡선은 "지금 보유수량을 과거에도 그대로 들고 있었다면"
이라는 가정으로 그린다. 원금선이 거의 평평한 이유가 이것이다.
미래에셋 거래내역(매수일·수량·단가)을 넣으면 실제 곡선으로 바뀐다.
